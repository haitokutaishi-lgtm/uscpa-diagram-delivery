今回のテーマ：Contingencies — loss contingency の計上と注記

USCPA FAR / Loss Contingencies（ASC 450-20）

偶発損失の分岐—probable ならいくら計上し、reasonably possible なら注記だけかを決める

試験では、訴訟などの loss contingency について、発生可能性と金額の見積り可否から「仕訳するか／注記だけか／何もしないか」を選びます。金額が範囲で示され、より確からしい一点がないときは、U.S. GAAP では範囲の下限を計上します。

① よくある悩み

試験でよく出る問題文の例

Helix Corp is a defendant in a lawsuit. Legal counsel concludes that a loss is probable. The estimated loss ranges from $240,000 to $600,000, and no amount within the range is a better estimate than any other amount. What amount of loss contingency should Helix accrue under U.S. GAAP?
  • A. $0
  • B. $240,000
  • C. $420,000
  • D. $600,000

このあと、たくまがつまずきやすい3つの考え方の誤りを対話で整理します。誤解①〜③は、試験でよく見かける間違いの番号です（選択肢 A〜D の記号とは別です）。

たくま（受講生）

誤解①　範囲があるから平均（中央値）を計上する

$240,000 と $600,000 の真ん中の $420,000（選択肢 C）を負債にすれば安全では？ 平均の方が「代表値」っぽいです。

あおい先生（コーチ）

U.S. GAAP（ASC 450-20）では、より確からしい一点がない範囲のとき、計上額は下限（minimum）です。中央値は IFRS 等で出てくる発想で、ここでは誤答の定番です。

たくま（受講生）

誤解②　最悪ケース（上限）を全部計上する

保守的に考えるなら、$600,000（選択肢 D）を全部負債にすべきでは？

あおい先生（コーチ）

上限まで計上するのは、原則としてより確からしい金額が上限である場合です。今回は「範囲内のどの額も同じ確からしさ」なので、計上は下限、追加で損失が増える可能性は注記で示します。

たくま（受講生）

誤解③　金額が確定していないから計上ゼロ

まだ裁判中で確定していないので、$0（選択肢 A）で注記だけ、という読み方は？

あおい先生（コーチ）

probable かつ reasonably estimable（ここでは範囲で見積り可能）なら、確定判決を待たずに計上します。$0 は、remote や、金額がまったく見積れないケースに近い誤答です。

② まず覚える3つ

loss contingency（ASC 450-20）の地図です。計算の前に、この3点を固定します。

  1. 1

    計上の条件は「probable ＋ 見積り可能」

    Probable（発生の可能性が高い）かつ金額が reasonably estimable なら、負債と損失を計上します。

  2. 2

    範囲で一点が特に確からしくない → 下限を計上

    範囲の下限を負債にし、上限までの追加可能性は注記します。中央値・上限の自動計上はしません。

  3. 3

    reasonably possible は注記、remote は原則なし

    Reasonably possible は原則として注記のみ。Remote は原則として計上も注記もしません（保証など例外は別論点）。

③ 会計の背景と仕訳

②の3つは、ここでの偶発損失をいつ負債にするかに対応します。

なぜこの処理か（会計の背景）

将来の支出が「ほぼ確実」で金額も見積れるなら、当期の費用と負債として先に認識します。まだ可能性が中程度なら、財務諸表の利用者に注記で知らせ、仕訳はしません。可能性が低いものは、原則として財務諸表に載せません（ASC 450-20）。

処理の順序： ①発生可能性を読む（probable / reasonably possible / remote） → ②金額が見積れるか確認 → ③probable なら計上額を決める（一点 or 範囲の下限） → ④必要なら注記

典型仕訳（Helix Corp 例）

期末に損失を計上（下限 $240,000）：

Dr. Litigation loss (or Loss from contingency)          240,000

    Cr. Contingent liability (or Accrued litigation)     240,000

注記では、追加で最大 $360,000（$600,000 − $240,000）まで損失が増える可能性がある旨を示します。中央値 $420,000 や上限 $600,000 をそのまま計上しません。

次の④ 計算の型は、この計上額 $240,000 を求める手順です。

④ 計算の型

Accrued loss contingency を求める5ステップです。

  1. 1英文の可能性語を読む：probable / reasonably possible / remote。
  2. 2金額が reasonably estimable か確認する（一点・範囲・見積り不可）。
  3. 3probable ＋ 見積り可能なら計上。reasonably possible なら注記のみ。remote なら原則なし。
  4. 4範囲で、no amount is a better estimate なら minimum を計上する。
  5. 5計上した下限を超えて上限までの追加可能性は、注記で示す（上限を仕訳しない）。

計算例（Helix Corp・上の試験問題と同じ数字）

項目金額メモ
発生可能性—probable
見積り範囲（下限）$240,000より確からしい一点なし → 計上
見積り範囲（上限）$600,000追加可能性は注記
中央値（誤答）$420,000U.S. GAAP の標準ではない
Accrued amount$240,000正解 B

⑤ 対話で整理

たくま：誤解①〜③のあと、範囲の下限と注記の関係を確認したいです。 あおい先生：Helix では計上 $240,000、上限までの追加は注記です。

たくま（受講生）

もし問題文が「$400,000 が最も確からしい」と書いていたら、どうなりますか？

あおい先生（コーチ）

そのときはより確からしい金額 $400,000 を計上します。下限ルールは、「範囲内のどの額も同じ確からしさ」のとき専用です。英文の best estimate / most likely を必ず探します。

たくま（受講生）

reasonably possible で金額が $100,000〜$300,000 と出ていたら、下限を計上しますか？

あおい先生（コーチ）

しません。計上の入口は probable です。reasonably possible は、性質と見積り（または見積り不能である旨）を注記します（②の3点目）。

⑥ 理解確認MC

オリジナル問題です。ASC 450-20 の考え方に沿って作成しています。第三者問題集からの転載はありません。

問題 1（計算 — accrued amount）

Helix Corp is a defendant in a lawsuit. Legal counsel concludes that a loss is probable. The estimated loss ranges from $240,000 to $600,000, and no amount within the range is a better estimate than any other amount. What amount of loss contingency should Helix accrue under U.S. GAAP?
  • A. $0
  • B. $240,000
  • C. $420,000
  • D. $600,000
解説を開く

probable ＋ 範囲で見積り可能 → 計上する。一点の better estimate がないので minimum $240,000。正解 B。

A＝誤解③。C＝中央値（誤解①）。D＝上限（誤解②）。

問題 2（概念 — recognition vs disclosure）

Under U.S. GAAP, a loss contingency is accrued when which conditions are met?
  • A. The loss is remote and the amount is reasonably estimable
  • B. The loss is reasonably possible, regardless of whether the amount can be estimated
  • C. Information available before the financial statements are issued indicates that it is probable a liability has been incurred, and the amount can be reasonably estimated
  • D. Management prefers to be conservative and therefore records the maximum possible loss
解説を開く

計上の条件は、probable かつ reasonably estimable です。正解 C。

A は remote。B は注記側。D は上限の自動計上（誤解②）で、GAAP の一般ルールではありません。

⑦ この図解のまとめ

  1. 偶発損失は、まず発生可能性を読む（probable / reasonably possible / remote）。
  2. 計上は probable ＋ reasonably estimable のとき（③の背景・仕訳）。
  3. 範囲で better estimate がないときは 下限を計上し、上限までの追加は注記（誤解①②）。
  4. 確定判決前でも、条件を満たせば計上する（誤解③）。
  5. reasonably possible は注記、remote は原則として計上も注記もしない。
  6. 仕訳：Dr. Litigation loss / Cr. Contingent liability（勘定名は問題文に合わせる）。
